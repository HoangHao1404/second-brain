# Personal Study Assistant

> [!info] Định nghĩa
> App cho phép sinh viên chat với tài liệu học tập của chính họ bằng AI. Người dùng có thể upload tài liệu (Word, PDF, ảnh ghi chú tay) và tương tác với AI để tóm tắt, tìm thông tin, ôn tập.

## Thông tin đã phân tích

> [!important] Target users
> Sinh viên không kỹ thuật

> [!warning] Pain point
> Có đống tài liệu (Word, PDF, ảnh ghi chú tay) nhưng ôn thi hoặc tìm lại thông tin rất mất công

> [!tip] Tính năng cốt lõi
> **Core value:** Upload tài liệu cá nhân → chat với AI để tóm tắt, tìm thông tin, ôn tập

> [!success] Lợi thế so với giải pháp hiện có
> - **ChatGPT:** Không có tài liệu riêng của người dùng — app này giải quyết bằng private data
> - **Obsidian:** Dành cho người kỹ thuật, sinh viên thông thường không dùng được

## Lộ trình làm việc từ đầu đến cuối về phía BA

### [[Giai đoạn 1 — Problem & Stakeholder Analysis]]
**Output:** Stakeholder map + Problem statement rõ ràng
Việc cần làm:
- [x] Xác định ai là stakeholder (không chỉ user)
- [x] Viết problem statement theo chuẩn (As a... I struggle with... because...)
- [x] Xác định success metric: thế nào là app "thành công"?

---

### [[Giai đoạn 2 — Requirements]]

**Output:** Functional Requirements + Non-functional Requirements

Việc cần làm:
- [x] Liệt kê tính năng theo nhóm (upload, chat, quản lý tài liệu...)
- [x] Phân loại Must-have / Should-have / Nice-to-have (MoSCoW)
- [x] Xác định các ràng buộc: bảo mật, hiệu năng, giới hạn file...

---

### [[Giai đoạn 3 — Use Case & User Flow]]

**Output:** Use case diagram + User flow cho các luồng chính

Việc cần làm:
- [x] Vẽ use case diagram
- [x] Viết use case specification cho 3-4 luồng cốt lõi
- [x] Xác định edge case và error flow

---

### [[Giai đoạn 4 — System Design (góc nhìn BA-level)]]

**Output:** Context diagram + Data flow + Entity mô tả sơ bộ

Việc cần làm:
- [x] Vẽ system context diagram (app của bạn tương tác với gì bên ngoài: AI API, storage...)
- [x] Mô tả luồng dữ liệu: tài liệu đi từ đâu đến đâu
- [x] Định nghĩa entity chính: User, Document, Conversation, Message...

---

### [[Giai đoạn 5 — Validation & Traceability]]

**Output:** Requirements traceability matrix (RTM) sơ bộ

Việc cần làm:
- [x] Đảm bảo mỗi requirement đều có use case tương ứng
- [x] Đảm bảo mỗi use case đều trace về một pain point thật
